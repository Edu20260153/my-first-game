# 6주차 실습 기록

## 사용한 에셋
- 지구 애니메이션 스프라이트: https://deep-fold.itch.io/pixel-planet-generator
- 배경음악(C_1): pixabay - Space Ambient(leberch)

## 사용한 AI 프롬프트 (요약)
1. **Q:** 이미지 스프라이트를 Base64로 디코딩해서 파이썬에서 애니메이션으로 재생하는 원리가 뭐야?
- 파이썬 재생:base64.b64decode()로 텍스트를 다시 바이트(Byte)로 되돌리는 것이지만 파이게임은 파일 경로에서 이미지를 읽는 데 익숙하므로, io.BytesIO()를 써서 이 바이트 데이터를 "가상의 파일"처럼 메모리에 올려 pygame.image.load()가 읽을 수 있게 속인다.
- 애니메이션:pygame.time.get_ticks()을 이용하고, "(index + 1) % 총 프레임 수"공식을 사용해서 각 그리드의 이미지를 그리고, 바꿔치기하는 원리이다.

2. **Q:** 파이썬(Pygame)에서 사운드를 무한 반복 재생하거나 한 번만 재생하고, 특정 시점에 사운드를 끄려면 어떻게 해야 해?
- 반복 재생: pygame.mixer.music 모듈을 사용, music.load('경로')로 불러오고, music.play(-1)에서 -1을 인자로 주면 무한 반복 재생됨.
- 한 번 재생: pygame.mixer.Sound('경로')로 객체를 만든 뒤, 필요할 때마다 sound_obj.play()를 호출하면 한 번만 재생된다.
- 특정 부분 종료: BGM을 끄고 싶다면 pygame.mixer.music.stop(), 재생 중인 특정 효과음만 끄고 싶다면 해당 객체에서 sound_obj.stop()을 호출하면 된다.

3. **Q:** 플레이어가 이동할 때 카메라가 부드럽게 플레이어를 따라다니게(추적하게) 하려면 어떤 원리를 써야 해?
- **선형 보간법(Lerp, Linear Interpolation)**을 사용해야 한다. 원리는 목표 카메라의 위치를 플레이어 좌표 - 화면 절반 크기(중앙 정렬)공식을 사용해서 자연스레 카메라가 플래이어를 따라다니도록 하는 효과를 만들수 있음.

4. **Q:** 배경의 반짝이는 파티클 레이어를 2개로 나눠서, 앞쪽은 빠르게 뒤쪽은 느리게 움직이는 패럴랙스(원근감) 효과의 원리와 구현 방법은 뭐야?
- 앞쪽 레이어와, 뒷쪽 레이어를 나누어 카메라의 좌표에 따라 이팩트의 위치를 따로 계산해 주면 된다.

## AI 답변에서 도움이 된 것
- 메모리 상에서 이미지를 변환하는 `io.BytesIO`의 원리와, 시간(Tick)에 따라 이미지를 교체하는 애니메이션의 기본 구조를 이해하게 되었습니다.
- BGM(`music`)과 효과음(`Sound`)을 분리해서 관리해야 제어가 쉽다는 점을 알게 되었습니다.
- 선형 보간법(Lerp)을 활용해 카메라가 플레이어를 딱딱하게 따라가지 않고 부드럽게 쫓아가는 수학적 원리가 큰 도움이 되었습니다.
- 카메라 이동량에 서로 다른 소수를 곱하는 것만으로 2D 화면에 공간감과 깊이를 부여하는 패럴랙스 스크롤링 기법을 적용할 수 있었습니다.

## AI 답변을 수정하거나 버린 것
- 카메라가 맵 밖으로 넘어가지 않도록 `max`와 `min`을 사용해 좌표를 제한하는 클램핑코드를 내 게임 맵 크기에 맞게 조정함.
- 배경 레이어의 이동 속도 수치를 게임의 템포에 맞춰 각각 `0.3`과 `0.05` 등으로 직접 변경하여 적용함.

## 적용 결과
- **잘 된 것:** 카메라가 부드럽게 이동하고, 배경 레이어가 속도 차이를 두며 움직여서 우주 공간의 깊이감이 확실하게 살아났습니다. 사운드도 상황에 맞게 정확히 재생 및 종료됩니다.
- **어려웠던 것:** Base64 문자열이 너무 길어서 코드를 관리하는 것과, 카메라 오프셋(offset) 좌표를 모든 객체 렌더링에 빼줘야 하는 부분을 놓치지 않고 적용하는 과정이 조금 헷갈렸습니다.
- **다음에 시도할 것:** 플레이어의 체력이 낮아졌을 때 BGM의 피치(Pitch)를 낮추거나 심장 박동 사운드를 추가해 긴장감을 높이는 효과를 구현해 보고 싶습니다.
