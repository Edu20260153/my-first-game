# 목표
키보드를 이용해서 화면의 파란 원을 이동시켜라!

# AI 대화 기록
* Q1: 현제 코드에서 키보드(W,A,S,D) 혹은 방향키(Up, left, Down, Rigth)를 눌렀을때 그 방향으로 SPEED만큼 이동하려면 어떻게 코드를 수정해야되?

- AI답변: pygame.key.get_pressed()를 사용하여 매 프레임마다 키 리스트를 받아 해당키가 눌렸는지를 if 조건문을 사용해 검사한 후 draw할때 좌표부분을 따로 Pos 빼어 선언한후 draw 전에 조건부 검사를 통해 좌표를 SPEED만큼 해당 방향으로 이동시키는 방법을 재안함.

- 적용: 성공

* Q2: clock Tick을 낮게 설정했더니 tick 60때보다 속도가 낮은데 tick을 fps개념으로 보고 tick이 다를때도 속도 자체자 유지되도록 하고싶은데 어떻게 코드를 수정해야되?

- AI답변: 현제 tick 방식을 시간기반 이동 방식(delta time, dt)으로 변경 dt = tick(FPS)/1000으로 계산하고 SPEED를 picel/s형식으로 수렴.

- 적용: 성공

* Q3: 좀더 현실적인 이동을 위해 현제 이동방식에 물리적인 요소(관성,가속,마찰)을 추가해 오브잭트가 좀더 부드럽게 움직일수 있도록 코드를 작성하고싶은데 어떻게 작성해야해?

- AI답변: 간단한 물리적 계산을 토데로 적용 가능한 방법을 알려줌

- 시행착오: 계산방법을 몰라서 자세한 공식과 현제 코드에서 어떻게 작성해야되는지 질문함.

- 최종해결: 마찰과 가속, 감속, 최고속도등 도입과 공식 등을 이해할수 있었음.

- 배운것: 키를 누를땐 가속도를 통해 최고 속도에 도달하고 키를 땔땐 마찰과 감속 수치 계산을 통해 감속한다는 것을 앎.

* Q4: 대각선으로 갈때 좀 빠르게 가는 느낌이 듦.

- AI답변: 대각선으로 갈땐 root(2)를 speed에 곱해 자련스럽게 가도록 하는 방법을 알려줌.

- 적용: 성공

- 배운것: 대각선에 갈땐 삼각함수를 활용해서 대각선의 속도를 계산한 것이 speed * root(2)라는것을 알게 됨

* Q5: 화면 경계를 벗어나지 못하도록 하는 방법을 알려줘.

- AI답변: 오브잭트의 좌표가 화변 경계 밖으로 나가려 할때 나간 좌표만큼 강제로 이동시켜 나가지 못하도록 코드를 작성함

- 적용: 성공

* Q6: 탄피를 추가해 탄피가 일정한 속도로 오브잭트 방향으로 쭉 이동해서 화면 경계 밖에 나갔을떄 사라지도록 하는 탄피요소를 추가하는 방법을 알려줘

- AI답변: 기존 코드에서 오브잭트 draw방식을 재사용해서 bullets 리스트를 만들어 총알 좌표 위치, 방향 등을 기록해 매 프래임마다 업데이트 하도록 하는 방식을 알려줌

- 적용: 성공

- 든 생각: 특별한 방법없이 리스트로 볼트를 새로 만든다는 반복작업이었다는것에 살짝 귀찮은 느낌이 있었음.

* Q7: 탄피의 충돌기능을 넣어 충돌했을때 Game Over를 화면 중간에 띄우고 게임을 멈췄으면 하는데 draw에서 서로의 충돌을 어떻게 감지해?

- AI답변: 따로 충돌 감지는 없고 좌표의 수지와 두 반지름을 더한 값이 두 위치의 차보다 크면 충돌됨이라 인식하도록함

- 든 생각: CPU를 너무 사용하는 방법이 아닐까 생각함.

- 조사: 다른 방법이 있지 않을까 AI에게 질문

- AI답변: 수치계산을 벗어날순 없지만 현제 계산을 최적화하는 방법을 재시해줌

- 배운것: 곱샘이나 실수 계산이 비용이 많이 든다는것을 알게됨.
